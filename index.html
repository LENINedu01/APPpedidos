1. Introducción: ¿Para qué sirve esta app?

Esta aplicación Java busca **modernizar** la toma de pedidos en restaurantes, permitiendo a los meseros en general realizar de formas mas practica y fluida la toma de pedidos asi como el seguimiento de pedidos en el día.

Beneficios Clave:

  • Eficiencia: Reducción de hasta un 30% en los tiempos de servicio.
  • Integración: Pedidos enviados directamente a la cocina vía API.
  • Rentabilidad: Aumento del consumo promedio por la facilidad de navegación del menú.

5. BD: Creación de Tabla Pedidos

-- Pseudocódigo SQL para la tabla principal
CREATE TABLE pedidos (
    pedido_id INT PRIMARY KEY AUTO_INCREMENT,
    mesa_id VARCHAR(50) NOT NULL,
    hora_pedido TIMESTAMP,
    total DECIMAL(10, 2) NOT NULL,
    estado VARCHAR(20) DEFAULT 'pendiente'
);

6. Conexión: Fetch de Productos

// funciones.js - Envío del Pedido
function submitOrder() {
    const payload = {
        mesa: mesaSeleccionada,
        items: cart,
        total: subtotal
    };

    fetch('https://api.restaurant.com/pedidos', {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify(payload)
    })
    .then(response => {
        if (response.ok) {
            console.log("Pedido enviado con éxito.");
        }
    });
}

4. Base de Datos: Esquema y Relaciones

Se requiere un esquema relacional sólido para manejar la concurrencia de pedidos y el inventario.

Tablas Principales

  • **Productos:** ID, nombre, precio, categoría, stock.
  • **Mesas:** ID, capacidad, estado (ocupada/libre).
  • **Usuarios:** ID, nombre, rol (cocina/camarero/admin).

Tablas Transaccionales

  • **Pedidos:** `pedido_id` (PK), `mesa_id` (FK), `hora_pedido`, `total`, `estado`.
  • **Detalle_Pedidos:** `detalle_id` (PK), `pedido_id` (FK), `producto_id` (FK), `cantidad`, `precio_unitario`.

5. Conexión: App/Web a BD (API REST)

API REST y Endpoints

Se utiliza una API RESTful como intermediario entre la aplicación (cliente) y la base de datos (servidor).

  • GET /api/productos: Obtener menú.
  • POST /api/pedidos: Enviar la nueva orden.
  • PATCH /api/pedidos/{id}: Actualizar estado.
